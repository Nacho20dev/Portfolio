Estructura del Sistema

El núcleo del proyecto separa totalmente las reglas de negocio de la infraestructura tecnológica. El Cliente consume la WebAPI (endpoints) y el Funcionario gestiona el Backoffice.

Cliente -> Endpoints

Consulta tarifas, realiza solicitudes y sigue envíos consumiendo la API.

Funcionario -> Backoffice

Administra ingresos, egresos y estados operativos desde el panel de control.

Flujo de Datos y Dependencias
Capa Presentación (WebAPI / MVC / Controladores)
Capa Aplicación (Casos de Uso / Interfaces UC)
Capa Dominio (Entidades / Interfaces Repo)
Capa Infraestructura (EF Core / SQL / Seguridad)

Seguridad Robusta

JWT stateless, Hasheo de contraseñas y control de roles para accesos seguros.

Entity Framework

Enfoque Code-First con SQL Server para una persistencia estructurada.

Inyección de Dependencias

Cumplimiento de principios SOLID (IoC) para garantizar un código testeable y mantenible.

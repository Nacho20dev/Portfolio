Clean Architecture & DDD

El núcleo del proyecto se basa en la separación total de preocupaciones. Las reglas de negocio (Casos de Uso) no conocen los detalles de la base de datos o la interfaz de usuario.

  • Dominio: Entidades puras y lógica core.
  • Aplicación: Interfaces de repositorio y orquestación de servicios.
  • Infraestructura: Persistencia con EF Core y seguridad.
  • WebAPI/MVC: Controladores desacoplados.

Seguridad Robusta

Implementación de Hasheo de contraseñas con salting y tokens JWT para una comunicación API segura.

Entity Framework

Inyección de Dependencias

Cumplimiento estricto del principio de Inversión de Dependencias (SOLID) para un código testeable.

Estructura de un Caso de Uso

public interface ICrearEnvioUC {
    void Ejecutar(EnvioDTO nuevoEnvio);
}

public class CrearEnvioUC : ICrearEnvioUC {
    private readonly IRepositorioEnvios _repo;
    
    public CrearEnvioUC(IRepositorioEnvios repo) {
        _repo = repo;
    }

    public void Ejecutar(EnvioDTO dto) {
        // Lógica de negocio validada antes de persistir
        _repo.Add(dto.MapToDomain());
    }
}

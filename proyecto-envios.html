LOGIVIP

Sistema de Gestión Logística con Arquitectura Distribuida (Web API + Cliente MVC)

.NET CORE 8 WEB API RESTFUL JWT AUTH ENTITY FRAMEWORK

Respaldo Documental

El sistema fue diseñado bajo un proceso de análisis riguroso, documentado en una memoria técnica de 140 páginas que detalla:

  • Modelo UML: Diagramas de clases (archivo .asta) con lógica de herencia para Envíos Comunes y Urgentes.
  • Trazabilidad: Implementación de etapas de seguimiento (Seguimientos) con comentarios y auditoría de empleados.
  • Persistencia: DER normalizado con registro de acciones para auditoría de Administradores (RF2).
Lógica de Negocio Crítica

Implementación de la funcionalidad de Finalización de Envío:

// Polimorfismo en el dominio
if (envio is Urgente urgente) {
  urgente.CalcularEficiencia(fechaEntrega);
}
envio.Estado = Estado.FINALIZADO;

Cálculo de eficiencia automático para entregas < 24hs (Requerimiento de Letra).

Funcionalidades Implementadas

ETAPA 1: GESTIÓN INTERNA (BACKOFFICE)
  • Login por Roles: Acceso diferenciado para Administradores y Funcionarios.
  • CRUD de Empleados: Gestión completa con auditoría obligatoria en base de datos.
  • Operativa de Carga: Alta de envíos y actualización de estados con comentarios de seguimiento.
  • Tracking Público: Búsqueda de detalles sin necesidad de autenticación.
ETAPA 2: CLIENTE HTTP & WEB API
  • Consumo Distribuido: Aplicación MVC independiente que consume la Web API vía HttpClient.
  • Seguridad JWT: Endpoints protegidos mediante Tokens para acciones del cliente.
  • Consultas Avanzadas: Filtrado de envíos por rango de fechas y búsqueda por contenido de comentarios (comportamiento "Contiene").
  • Perfil de Usuario: Cambio de contraseña con validación de credencial actual.
